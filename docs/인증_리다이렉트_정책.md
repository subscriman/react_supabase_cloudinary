# 인증 리다이렉트 정책 (웹/앱 공통)

기준일: 2026-04-02

## 1. 목표

- 웹 MVP와 2차 React Native 앱에서 동일한 로그인 흐름을 재사용한다.
- Supabase Auth Provider별 redirect URL을 일관된 규칙으로 관리한다.

## 2. 웹 기준 정책

- 웹 OAuth/이메일 링크 callback 경로는 고정: `/auth/callback`
- 로그인 완료 후 기본 이동 경로: `/mypage`
- next 경로는 query `next`로 전달하고, 내부 경로(`/...`)만 허용한다.
- 최종 redirect URL 생성은 `web/lib/auth-redirect.ts`에서 단일 책임으로 관리한다.

## 3. Supabase Dashboard 설정값

Supabase Auth > URL Configuration:

- Site URL: `https://arttomato-web.vercel.app`
- Redirect URLs:
  - `https://arttomato-web.vercel.app/auth/callback`
  - `http://localhost:3000/auth/callback`

소셜 Provider 콘솔(Google/Kakao/Naver)에도 동일 callback URL을 등록한다.

## 4. 앱(2차) 확장 정책

- 앱 callback 템플릿: `arttomato://auth/callback`
- 웹과 앱 모두 `next` 파라미터를 공통으로 사용한다.
- 앱은 deep link를 받고, 동일한 세션 확인 로직 후 목적지 화면으로 이동한다.

## 5. Vercel 환경 변수

- `NEXT_PUBLIC_SITE_URL`는 실제 배포 도메인으로 고정한다.
- 현재 Production/Development는 등록 완료.
- Preview는 프로젝트 Git 연동 이후 브랜치 기준으로 추가한다.
