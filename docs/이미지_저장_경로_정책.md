# 이미지 저장 경로 정책 (Supabase Storage)

기준일: 2026-04-02

## 1. 버킷 구성

- `exhibition-assets`: 전시 포스터/커버
- `review-images`: 리뷰 첨부 이미지
- `profile-avatars`: 사용자 아바타

## 2. 경로 규칙

- 전시 포스터:
  - `posters/{sourceSiteKey}/{yyyy}/{mm}/{exhibitionSlug}.webp`
- 리뷰 이미지:
  - `reviews/{userId}/{reviewId}/image-{index}.webp`
- 프로필 아바타:
  - `avatars/{userId}/profile.webp`

## 3. 파일명 규칙

- 영문 소문자 + 숫자 + `-`, `_`만 허용
- 공백/특수문자는 `-`로 치환
- 연속 `-`는 하나로 축약
- 확장자 기본값은 `webp`

코드 헬퍼:

- `web/lib/storage-policy.ts`
  - `buildExhibitionPosterPath`
  - `buildReviewImagePath`
  - `buildAvatarPath`

## 4. 공개 범위 원칙

- `exhibition-assets`: public read
- `review-images`: private (작성자/관리자만 읽기, 공개 URL은 서명 방식)
- `profile-avatars`: public read

## 5. 운영 원칙

- 업로드 전 리사이즈/압축 후 저장 (권장 최대 변: 1600px)
- 원본 이미지 재업로드 시 동일 경로 overwrite 가능
- 삭제 정책:
  - 리뷰 삭제 시 `review-images` 경로 정리
  - 계정 삭제 시 `profile-avatars` 경로 정리
