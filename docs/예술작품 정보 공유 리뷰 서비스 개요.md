# ArtTomato 서비스 개요

## 1. 서비스 한 줄 소개

ArtTomato는 지금 열리고 있는 전시를 한눈에 찾고, 실제 관람객의 별점과 짧은 후기를 확인할 수 있는 전시 리뷰 서비스다.

영화는 네이버 영화나 IMDb처럼 대중 평가가 활발하지만, 전시는 여전히 공식 소개 페이지나 블로그 후기 위주로 정보가 흩어져 있다. ArtTomato는 이 틈을 해결하기 위해 전시 정보와 사용자 리뷰를 한곳에 모은다.

## 2. 해결하려는 문제

- 전시 정보가 기관별 사이트에 흩어져 있어 한 번에 찾기 어렵다.
- 공식 소개만으로는 실제 관람 만족도, 혼잡도, 관람 시간 같은 체감 정보를 알기 어렵다.
- 블로그 후기나 SNS 반응은 검색에 시간이 걸리고 품질 편차가 크다.
- 예술 전시는 기간이 짧고 일회성이 강해, 영화처럼 대규모 영구 DB보다 "현재 열리는 전시" 중심의 탐색 경험이 더 중요하다.

## 3. 핵심 가치 제안

- 사용자는 현재 진행 중인 전시를 빠르게 찾을 수 있다.
- 공식 정보와 실제 관람객 후기를 함께 볼 수 있다.
- 단순 별점뿐 아니라 관람 팁 중심의 짧은 리뷰를 통해 방문 결정을 쉽게 할 수 있다.
- 전시명뿐 아니라 작가명, 사조, 장르, 공간명을 기준으로 탐색할 수 있다.

## 4. 서비스 방향

ArtTomato는 처음부터 "예술작품 전체를 다루는 거대한 플랫폼"을 목표로 하지 않는다. 1차 목표는 전시 중심의 탐색 및 리뷰 서비스다.

초기 범위는 다음과 같이 제한한다.

- 메인 카테고리는 전시와 미술관/박물관 기획전
- 뮤지컬, 연극, 공연은 2차 확장 범위로 보류
- 전국 전체를 한 번에 수집하지 않고, 주요 기관 중심으로 시작
- 과거 전시 아카이브보다 현재 진행 중이거나 곧 열릴 전시에 집중

## 5. 핵심 타깃 사용자

### 1차 타깃

- 주말에 갈 만한 전시를 찾는 20~40대 일반 관람객
- 미술 전공자가 아니어도 가볍게 전시를 즐기는 사용자
- 데이트, 혼자 나들이, 가족 관람을 위해 전시를 찾는 사용자

### 2차 타깃

- 특정 작가나 사조를 따라 전시를 찾는 미술 애호가
- 전시 정보를 빠르게 모니터링하고 싶은 문화생활 관심층

## 6. MVP 목표

MVP의 목적은 "전시를 찾고, 실제 관람 후기를 보고, 나도 쉽게 리뷰를 남길 수 있는가"를 검증하는 것이다.

초기 MVP에서는 아래 세 가지가 가장 중요하다.

- 전시 정보가 안정적으로 올라오는가
- 사용자가 리뷰를 실제로 남기는가
- 전시 상세 페이지가 방문 결정에 도움이 되는가

## 7. MVP 핵심 기능

### 1) 전시 정보 수집 및 등록

- 국립현대미술관, 예술의전당, 서울시립미술관 등 주요 기관의 전시 정보를 수집한다.
- 수집 항목은 전시명, 전시장소, 전시 기간, 운영 시간, 포스터 이미지, 소개 문구, 예매 링크, 공식 링크를 기본으로 한다.
- 자동 수집 결과는 바로 공개하지 않고 관리자 검수 후 게시한다.
- 초기에는 "완전 자동 게시"보다 "반자동 수집 + 관리자 승인" 방식을 우선한다.

### 2) 전시 목록 및 탐색

- 메인 화면에서 현재 진행 중인 전시를 포스터 그리드 형태로 보여준다.
- 사용자는 진행 중, 예정, 종료 임박, 지역, 장르, 공간 기준으로 필터링할 수 있다.
- 검색은 전시명, 작가명, 공간명, 태그 기준으로 지원한다.

### 3) 전시 상세 페이지

- 전시 기본 정보
- 전시 소개
- 전시 기간 및 운영 시간
- 장소 정보 및 공식 링크
- 평균 평점
- 사용자 리뷰 목록
- 관련 태그: 작가, 사조, 스타일, 장르

### 4) 평점 및 리뷰

- 5점 만점, 0.5점 단위 별점
- 한줄 총평 작성
- 아래 항목을 선택형으로 추가해 리뷰 효용을 높인다.
  - 관람 추천 대상: 혼자, 친구와, 데이트, 가족
  - 예상 관람 시간: 30분, 1시간, 2시간 이상
  - 재방문 의사: 있음, 보통, 없음
  - 혼잡도: 여유, 보통, 혼잡

짧은 리뷰 중심으로 시작하되, 향후에는 사진 첨부나 긴 리뷰를 확장 기능으로 추가할 수 있다.

### 5) 회원 기능

- 이메일 로그인
- 소셜 로그인: 카카오, 네이버, 구글
- 내가 작성한 리뷰 목록 확인
- 찜한 전시 저장 기능은 2차 기능으로 고려

### 6) 관리자 기능

- 자동 수집된 전시 임시 저장
- 관리자 검토 후 게시 / 반려 / 수정
- 전시 중복 확인
- 태그 정리 및 대표 이미지 보정

### 이후 확장 방향

- 지역 박물관 및 미술관 확대
- 민간 갤러리 및 화랑 일정 수집
- 전시 예매 플랫폼 또는 전시 큐레이션 사이트 연동 검토

### 수집 원칙

- 공식 사이트 기반 정보 수집을 우선한다.
- 수집 결과는 AI로 요약/정제할 수 있지만, 최종 게시는 관리자 승인 후 진행한다.
- AI 정제 단계에서는 `.env`에 설정된 OpenAI 호환 LLM API를 사용해 전시명, 장소명, 기간, 소개 문구, 태그 후보를 정리한다.
- 현재 LLM 연동은 `LLM_API_KEY`, `LLM_BASE_URL`, `LLM_MODEL_NAME` 환경 변수를 기준으로 구성하며, 기본 모델은 `qwen-plus`를 사용한다.
- 초기에는 외부 블로그 후기 자동 링크 수집보다, 서비스 내부 리뷰 축적을 우선 목표로 한다.
- 외부 후기 링크는 필요할 경우 관리자 큐레이션 방식으로 제한적으로 운영한다.

## 9. 데이터 구조 방향

주요 엔티티는 아래와 같다.

- exhibitions: 전시 기본 정보
- venues: 전시장/미술관/박물관 정보
- artists: 작가 정보
- tags: 작가명, 사조, 장르, 스타일 등 탐색용 태그
- exhibition_tags: 전시와 태그 연결
- reviews: 사용자 리뷰 및 평점
- profiles: 사용자 프로필
- ingestion_jobs: 수집 작업 이력

이 구조를 사용하면 "고흐" 같은 키워드로 검색했을 때 전시명뿐 아니라 관련 작가 태그가 붙은 전시까지 함께 보여줄 수 있다.

## 10. 사용자 경험 방향

### 메인 페이지

- 전시 포스터 중심의 시각적 그리드
- 현재 진행 중인 전시 우선 노출
- 모바일에서도 빠르게 훑어볼 수 있는 카드형 구조

### 상세 페이지

- 포스터, 제목, 기간, 장소, 평점을 상단에서 바로 확인
- 아래에서 소개와 리뷰를 자연스럽게 이어서 읽는 구조
- "실제 방문 전 참고할 정보"가 눈에 잘 들어오도록 구성

### 디자인 방향

- 모던하고 미니멀한 UI
- 전시 포스터가 주인공이 되도록 여백 중심 레이아웃 사용
- 과한 장식보다 포스터 이미지, 타이포그래피, 카드 정렬감에 집중

## 11. 운영 흐름

### 전시 등록 흐름

1. 매일 정해진 시간에 수집 작업 실행
2. 대상 사이트에서 전시 정보 수집
3. AI 또는 규칙 기반 로직으로 필드 정리
4. 관리자 화면에 임시 등록
5. 관리자 검토 후 게시 또는 반려

### 리뷰 생성 흐름

1. 사용자가 전시 상세 페이지 방문
2. 별점과 한줄 총평 작성
3. 선택형 관람 정보 입력
4. 리뷰 저장 후 평균 평점 반영

### 리뷰 운영 정책 초안

- 리뷰는 기본 공개를 원칙으로 하되, 운영자가 부적절 콘텐츠를 확인하면 `is_hidden` 처리한다.
- 작성자는 본인 리뷰만 수정/삭제 가능하며, 한 전시당 1개의 리뷰만 작성한다.
- 신고 기능은 2차 단계에서 도입하며, MVP 단계에서는 관리자 검토 기반의 수동 숨김 정책으로 운영한다.
- 신고 기능 도입 시 누적 신고 수, 신고 사유, 처리 상태를 별도 테이블로 관리해 추적 가능하게 설계한다.

## 12. 기술 구현 방향

기술 스택은 [arttomato_stack.md](/Users/shin/workspace/ArtTomato/docs/arttomato_stack.md)를 기준으로 진행한다.

### 웹

- Next.js Pages Router 기반으로 구현 (1차 MVP)
- SEO 대응이 가능한 전시 상세 페이지 구성
- 반응형으로 PC와 모바일 웹 동시 대응

### 백엔드 및 데이터

- Supabase를 활용해 PostgreSQL, Auth, Storage를 통합 운영
- 전시, 리뷰, 사용자 정보를 관계형 구조로 관리
- 이미지와 리뷰 사진은 Supabase Storage에 저장

### AI 정제

- 전시 수집 파이프라인에서 OpenAI 호환 LLM API를 사용해 수집 결과를 정규화한다.
- 주요 용도는 필드 정리, 소개 문구 요약, 태그 후보 생성, 중복 후보 판별 보조다.
- LLM 설정은 `.env`의 `LLM_API_KEY`, `LLM_BASE_URL`, `LLM_MODEL_NAME`을 사용한다.
- 현재 기준 모델은 DashScope 호환 엔드포인트의 `qwen-plus`다.

### 배포

- 웹은 Vercel에 배포
- GitHub 연동을 통해 자동 배포 구성
- MVP 단계에서는 비용을 최소화하는 구조를 유지

### 모바일 확장

- 웹 서비스 안정화 이후 React Native(Expo) 앱 개발
- Supabase 기반 로직과 데이터 구조를 최대한 재사용
- 1차 웹 단계에서 데이터 조회/저장 로직을 서비스 계층으로 분리해 앱 재사용성을 확보
- 1차 웹 단계에서 식별자, 날짜/시간, 이미지 경로, 에러 코드 규칙을 표준화
- 1차 웹 단계에서 인증 및 소셜 로그인 redirect 정책을 앱 확장까지 고려해 설계

## 15. 결론

ArtTomato의 핵심은 "전문가 중심 예술 정보"를 "일반 관람객 중심 전시 탐색 경험"으로 바꾸는 데 있다.

처음부터 모든 예술 분야를 다루기보다, 현재 진행 중인 전시를 중심으로 정보와 리뷰를 축적하는 서비스로 시작하는 것이 현실적이다. 이 방향이라면 적은 비용으로도 MVP를 빠르게 만들 수 있고, 이후 공연, 뮤지컬, 앱 확장까지 자연스럽게 이어갈 수 있다.
