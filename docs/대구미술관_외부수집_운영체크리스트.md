# 대구미술관 외부 수집 운영 체크리스트

기준일: 2026-04-03

## 목적

- 기본 수집(11개 사이트)과 분리해서 `daegu-art`를 외부 환경에서만 검증/실행한다.
- 로컬 환경에서 발생한 DNS/연결 이슈 영향을 운영 배치와 분리한다.

## 사전 조건

- [ ] 실행 환경에서 `daeguartmuseum.or.kr` DNS 해석 가능
- [ ] `NEXT_PUBLIC_SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` 설정 완료
- [ ] `LLM_API_KEY`, `LLM_BASE_URL`, `LLM_MODEL_NAME` 설정 완료
- [ ] `INGESTION_ENABLE_DAEGU_ART=true` 설정
- [ ] Supabase Edge Function 배포
  - `npm run supabase:functions:deploy:daegu-test`

## 1차 점검 (연결/파싱)

- [ ] Edge Function 연결 테스트
  - `npm run supabase:functions:invoke:daegu-test`
- [ ] Edge 경유 감사 실행
  - `npm run ingest:daegu:audit:edge`
- [ ] 리포트에서 `daegu-art` 상태가 `ok`인지 확인

## 2차 점검 (실행 시뮬레이션)

- [ ] Edge 경유 dry-run 실행
  - `npm run ingest:daegu:dry-run:edge`
- [ ] 로그에서 `raw`가 0보다 큰지 확인
- [ ] `fetch failed`/`connect timeout` 재발 여부 확인

## 3차 운영 반영

- [ ] 외부 배치 스케줄에 `daegu-art` 전용 실행 추가
- [ ] 실패 시 알림/로그 확인 경로 지정
- [ ] 1주일 관찰 후 지속 운영 여부 결정

## 권장 운영안

- 기본 배치: 11개 사이트 유지
- 보조 배치: `daegu-art` 단독 배치(외부 워커)
