# 로그 파일 문제 해결 가이드

## 문제 상황
- '테스트 로그 생성' 후 '로그 확인'을 눌렀을 때 "로그 파일이 존재하지 않습니다" 메시지 발생
- 로그가 `Documents/subscri` 폴더에 저장되어야 함

## 해결된 사항

### 1. 이중 로그 시스템 구현
- **react-native-logs**: 기본 로깅 라이브러리
- **수동 파일 작성**: RNFS를 사용한 직접 파일 작성 (백업)

### 2. 로그 디렉토리 및 파일 경로
```
저장 경로: /data/data/com.subscri.manager/files/Documents/subscri/
파일명: app-YYYY-MM-DD.log (예: app-2026-01-06.log)
```

### 3. 수정된 기능들

#### LoggerService 개선사항
- `ensureLogFile()`: 로그 파일이 없으면 자동 생성
- `writeLogManually()`: RNFS를 사용한 직접 파일 작성
- 모든 로그 메서드에서 이중 저장 (react-native-logs + 수동 작성)

#### 새로운 테스트 화면
- `LogTestScreen`: 로그 시스템 테스트 전용 화면
- 디렉토리 확인, 파일 생성, 로그 내용 확인 기능

## 테스트 방법

### 1. 앱에서 테스트
1. 홈 화면에서 "🔧 로그 테스트" 버튼 클릭 (개발 모드에서만 표시)
2. "디렉토리 수동 생성" → "테스트 파일 생성" → "테스트 로그 생성" 순서로 실행
3. "로그 디렉토리 확인" → "로그 내용 확인"으로 결과 확인

### 2. 파일 시스템에서 확인
```bash
# ADB를 통한 확인
adb shell
cd /data/data/com.subscri.manager/files/Documents/subscri/
ls -la
cat app-2026-01-06.log
```

### 3. Android 파일 관리자에서 확인
- 내부 저장소 > Android > data > com.subscri.manager > files > Documents > subscri

## 로그 파일 구조
```
[2026-01-06T13:08:15.123Z] [INFO] 테스트 로그 생성 시작
[2026-01-06T13:08:15.124Z] [DEBUG] 디버그 테스트 메시지
[2026-01-06T13:08:15.125Z] [WARN] 경고 테스트 메시지
[2026-01-06T13:08:15.126Z] [ERROR] 에러 테스트 메시지
```

## 문제 해결 단계

### 1단계: 권한 확인
- 앱이 내부 저장소 접근 권한을 가지고 있는지 확인
- Documents 폴더 접근 가능한지 확인

### 2단계: 디렉토리 생성
- LogTestScreen에서 "디렉토리 수동 생성" 실행
- 성공 메시지 확인

### 3단계: 파일 생성 테스트
- "테스트 파일 생성" 실행
- 파일이 생성되는지 확인

### 4단계: 로그 작성 테스트
- "테스트 로그 생성" 실행
- "로그 내용 확인"으로 내용 확인

## 추가 개선사항

### 1. 로그 레벨 설정
- Debug 모드: 모든 로그 저장
- Release 모드: Info 이상만 저장

### 2. 파일 관리
- 최대 7일치 로그 파일 보관
- 오래된 파일 자동 삭제

### 3. 에러 처리
- 파일 작성 실패 시 콘솔 로그로 대체
- 권한 문제 시 사용자에게 알림
