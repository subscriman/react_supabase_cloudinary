# 템플릿 시작 후 치환 체크리스트

이 문서는 이 저장소를 템플릿으로 사용해 새 프로젝트를 시작할 때, 반드시 교체하거나 다시 연결해야 하는 항목만 빠르게 점검하기 위한 체크리스트입니다.

## 0. 이 템플릿의 기본 예제값

아래 값은 템플릿 기본값입니다. 새 프로젝트에서는 대부분 교체 대상입니다.

- [ ] Android package: `com.subscri.manager`
- [ ] iOS project/target: `SubscriptionManager`
- [ ] 모바일 앱 표시명: `구독 관리`
- [ ] 루트 패키지 이름: `subscription-manager`
- [ ] 모바일 패키지 이름: `subscription-manager-mobile`
- [ ] 웹 패키지 이름: `subscription-manager-admin`
- [ ] GitHub 저장소 예시: `react_supabase_cloudinary`
- [ ] Netlify 사이트 예시: `subman-kiro-admin`
- [ ] Cloudinary upload preset 예시: `subscription_manager`
- [ ] Android 로그 폴더명 예시: `subscri`

## 1. 템플릿 복사 방식

- [ ] 새 프로젝트 폴더는 템플릿을 복사하되 `.git`, `node_modules`, `mobile/node_modules`, `mobile/ios/Pods`, `mobile/android/.gradle`, `mobile/.gradle`, `mobile/android/build`, `mobile/android/app/build`, `web/.next`, `.netlify`는 제외
- [ ] 새 프로젝트는 별도 Git 저장소로 시작할지, 기존 저장소를 유지할지 먼저 결정
- [ ] 복사 후 `npm install`, `cd mobile && npm install`, `cd mobile/ios && pod install`을 다시 실행

## 2. 프로젝트 이름 치환

- [ ] 루트 `package.json`의 프로젝트명/설명 점검
- [ ] `mobile/package.json`의 앱 이름 점검
- [ ] `web/package.json`의 웹 프로젝트명 점검
- [ ] `mobile/app.json`의 `name`, `displayName` 점검
- [ ] `README.md`의 프로젝트명, 저장소 URL, 설명 교체
- [ ] Netlify 사이트명이 바뀌면 `netlify.toml`과 Netlify 연결 상태도 다시 점검

## 3. Android 치환 항목

- [ ] `mobile/android/app/build.gradle`의 `namespace` 교체
- [ ] `mobile/android/app/build.gradle`의 `applicationId` 교체
- [ ] `mobile/android/app/src/main/res/values/strings.xml`의 앱 이름 교체
- [ ] `mobile/android/settings.gradle.kts`의 `rootProject.name` 점검
- [ ] `mobile/android/app/src/main/java/com/subscri/manager/` 패키지 경로를 새 패키지명에 맞게 이동
- [ ] `MainActivity` / `MainApplication` 상단 `package ...` 선언 교체
- [ ] Firebase를 쓰면 `mobile/android/app/google-services.json`을 새 앱 기준으로 다시 발급
- [ ] Google Maps를 쓰면 `GOOGLE_MAPS_API_KEY`를 새 프로젝트 키로 재설정

찾아서 치환할 값:

- [ ] `com.subscri.manager`
- [ ] `subscription-manager-mobile`
- [ ] `subscri`
