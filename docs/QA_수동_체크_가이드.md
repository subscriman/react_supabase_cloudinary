# ArtTomato QA 수동 체크 가이드

자동 점검(`npm run qa:smoke-web`)으로 커버되지 않는 항목을 수동으로 확인하기 위한 문서다.

## 0. 자동 보조 점검 (신규)

아래 커맨드는 수동 항목 중 핵심 7가지를 API/DB 기준으로 자동 검증한다.

```bash
node scripts/qa-manual-flows.js
```

점검 범위:

- 전시 시작 알림 추가/중복 차단/해제
- 찜 추가/중복 차단/해제
- 외부 후기 큐레이션 생성/수정/삭제
- 리뷰 작성/중복 차단/수정/삭제 (사진 경로 저장 포함)
- 관리자 승인(승인/반려/보류)
- 중복 전시 병합(hidden 처리)
- 수집 실패 로그 생성 + 복구 실행

주의:

- 이 스크립트는 QA용 임시 전시/장소를 생성 후 자동 정리한다.
- 수집 복구는 기본적으로 live 실행을 시도하며, 네트워크 장애 시 시뮬레이션 fallback이 동작할 수 있다.

## 1. 사전 준비

- 테스트 대상 URL: `https://arttomato-web.vercel.app` (또는 최신 Preview URL)
- 일반 사용자 계정 1개, 관리자 계정 1개 준비 (`npm run supabase:users`로 계정 존재 확인)
- `totalUsers=0`이면 `/auth`에서 이메일 로그인 1회 완료 후 다시 `npm run supabase:users` 확인
- 관리자 계정 승격 필요 시:

```bash
npm run supabase:grant-admin -- --email=you@example.com
```

## 2. 로그인 흐름 점검

- [ ] `/auth` 진입 시 이메일 로그인 영역 노출
- [ ] `/api/auth/providers` 기준으로 활성화된 소셜 버튼만 노출 (비활성 공급자는 숨김 또는 안내 문구 노출)
- [ ] 이메일 로그인 링크 수신 후 `/auth/callback` 경유하여 `/mypage` 진입
- [ ] 소셜 로그인 완료 후 `/mypage` 진입
- [ ] 로그아웃 후 홈(`/`) 복귀

## 3. 리뷰 작성 흐름 점검

대상: 공개 상태 전시 상세(`/exhibitions/[slug]`)

- [ ] 리뷰 작성: 별점 + 한줄평 + 긴 리뷰(선택) + 선택 필드 저장
- [ ] 리뷰 사진 첨부: JPG/PNG/WEBP 업로드 후 상세/마이페이지 노출
- [ ] 중복 작성 차단: 동일 계정으로 같은 전시에 재작성 시 충돌 메시지 확인
- [ ] 리뷰 수정: 기존 리뷰 내용 변경 후 반영 확인
- [ ] 리뷰 삭제: 삭제 후 목록/평균 평점 반영 확인

## 4. 관리자 승인 흐름 점검

대상: `/admin/exhibitions`

- [ ] 비관리자 계정 접근 시 권한 없음 메시지 확인
- [ ] 관리자 계정 접근 시 검수 대기 목록 노출
- [ ] 승인/반려/보류 각각 1건 실행 후 상태 변경 확인
- [ ] `/admin/ingestion-jobs`에서 수집 로그 목록 확인
- [ ] `/admin/duplicates`에서 중복 후보 목록과 병합 동작 확인

## 5. 수집 실패 복구 흐름 점검

실패 케이스 강제 생성(예시):

```bash
npm run ingest:run -- --site=unknown-site
```

확인 항목:

- [ ] 수집 작업이 실패 로그로 기록되는지 확인
- [ ] 관리자 화면(`/admin/ingestion-jobs`)에서 실패 사유 노출 확인
- [ ] 정상 사이트 재실행(`--site=mmca --limit=1`) 후 복구 확인

## 6. 중복 전시 처리 흐름 점검

- [ ] 제목/장소/기간이 동일한 전시 2건 준비
- [ ] `/admin/duplicates`에서 중복 후보로 탐지되는지 확인
- [ ] 병합 처리 후 대표 전시 1건만 공개 상태 유지되는지 확인
- [ ] 숨김 처리된 레코드가 공개 목록에서 제외되는지 확인

## 7. 완료 처리 기준

아래 4개가 모두 통과되면 체크리스트 14번 잔여 항목을 완료로 전환한다.

1. 로그인(이메일 + 최소 1개 소셜) 실제 성공
2. 리뷰 작성/수정/삭제 실동작 확인
3. 관리자 승인/반려/보류 실동작 확인
4. 실패 복구/중복 처리 실동작 확인
