# ArtTomato 전시 아카이브 확장 전략

기준일: 2026-04-03  
범위: 2차 개발 항목 `전시 아카이브 확장 전략 검토`

## 1. 목적

- 현재 중심(진행/예정 전시) 탐색 경험을 유지하면서, 종료 전시를 장기적으로 검색 가능한 아카이브로 확장한다.
- 데이터 적재량 증가 시에도 목록/검색 성능과 운영 비용을 통제한다.

## 2. 확장 원칙

- 기본 홈 화면은 계속 `ongoing/upcoming` 중심으로 유지한다.
- 종료 전시는 숨기지 않고 `ended`로 관리하되, 아카이브 전용 탐색 경로를 분리한다.
- 관리자 승인 정책은 종료 전시에도 동일하게 적용한다.
- 리뷰/평점은 전시 종료 후에도 보존한다.

## 3. 정보 구조(IA)

1. 웹 내 아카이브 진입
- `/archive` 또는 목록 필터 `status=ended`를 명시적 노출
- 기간(연/월), 지역, 태그, 기관 필터를 우선 제공

2. 상세 페이지 유지
- 기존 `/exhibitions/[slug]` 구조를 그대로 사용
- 종료 배지와 종료일 기준 정렬/추천 블록만 추가

3. 내부 링크 전략
- 종료 전시의 태그/작가/기관 링크를 통해 관련 종료 전시를 연쇄 탐색 가능하게 구성

## 4. 데이터/스키마 전략

1. 상태 전환 자동화
- `end_date` 지난 전시는 배치로 `ended` 상태 동기화
- `published_at`은 유지해 공개 기록을 보존

2. 검색 성능
- `status`, `end_date`, `published_at` 복합 인덱스 검토
- 장기적으로는 `ended` 전용 materialized view 또는 검색 테이블 분리

3. 미디어/스토리지
- 리뷰 이미지는 현재 정책(`review-images`) 유지
- 장기 비용 증가 시 썸네일/원본 분리 및 보존기간 정책 수립

## 5. 롤아웃 단계

1단계 (빠른 적용)
- 기존 목록에 `ended` 필터 노출 강화
- 종료 전시 정렬(최신 종료순)과 빈 상태 UX 개선

2단계 (아카이브 페이지)
- `/archive` 전용 페이지 구현
- 연도/기관 필터 추가

3단계 (품질 고도화)
- 아카이브 전시 큐레이션(월별 하이라이트)
- 외부 후기/내부 리뷰 기반 추천 카드

## 6. 운영 지표

- `archive_page_view` (아카이브 진입 수)
- `archive_filter_used` (필터 사용률)
- 종료 전시 상세 클릭률(아카이브 → 상세 전환율)
- 종료 전시 리뷰 작성률

## 7. 리스크 및 대응

- 리스크: 종료 전시 데이터 급증으로 목록 쿼리 지연
  - 대응: `ended` 전용 조회 경로/인덱스/페이지네이션 분리

- 리스크: 오래된 링크/이미지 소실
  - 대응: 정기 링크 헬스체크 + 깨진 링크 관리자 보정 큐

- 리스크: 홈 피드가 아카이브 콘텐츠에 잠식
  - 대응: 홈은 진행/예정 고정, 아카이브는 별도 탭 분리 유지

## 8. 구현 우선순위 제안

1. `ended` 탐색 UX 정리(필터/정렬/배지)
2. 아카이브 전용 페이지 추가
3. 아카이브 지표 이벤트 추가
4. 성능 최적화(인덱스/전용 뷰)
