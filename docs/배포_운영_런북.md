## 2. Supabase 운영 체크

자동 점검 스크립트:

```bash
npm run supabase:auth-check
npm run supabase:admin-check
npm run supabase:users
npm run supabase:rls-check
```

인증 공급자 체크(`supabase:auth-check`)는 `email/google/kakao/naver` 상태를 반환한다.
`--strict` 옵션을 붙이면 하나라도 비활성화 시 실패(exit 1)한다.

RLS 재점검 쿼리:

```sql
select schemaname, tablename, rowsecurity
from pg_tables
where schemaname = 'public'
  and tablename in ('exhibitions', 'reviews', 'profiles', 'source_sites', 'ingestion_jobs');
```

정책 점검:

```sql
select schemaname, tablename, policyname, permissive, roles, cmd
from pg_policies
where schemaname = 'public'
order by tablename, policyname;
```

최근 자동 점검 결과(2026-04-02):

- auth provider: `email=true`, `google=false`, `kakao=false`, `naver=false`
- admin profile count: `1`
- auth user count: `1`
- anon 기준 비공개 전시(`pending_review`, `hidden`, `rejected`) 조회: `0`
- anon 기준 `source_sites` 조회: `0`
- anon 기준 `ingestion_jobs` 조회: `0`
- anon 기준 `is_hidden=true` 리뷰 조회: `0`

## 3. 관리자 계정 생성

자동 승격 스크립트:

```bash
npm run supabase:grant-admin -- --email=you@example.com
```

또는:

```bash
npm run supabase:grant-admin -- --user-id=YOUR_AUTH_USER_UUID
```

현재 관리자 계정 수 확인:

```bash
npm run supabase:admin-check
npm run supabase:users
```

1. 대상 계정으로 로그인
2. `profiles` 생성 확인
3. SQL Editor 실행

```sql
update public.profiles
set role = 'admin'
where id = 'YOUR_AUTH_USER_UUID';
```

## 4. 수집 스케줄 운영 결정

MVP 운영안:

- 일 1회 정기 수집 (KST 06:00)
- 수동 재실행 허용 (관리자)
- 실패 시 다음 스케줄에서 자동 재시도
- 수집 결과는 `pending_review`로 적재 후 관리자 승인
- 전시 시작 알림 배치(일 1회, KST 09:00) 실행

관리자 수동 실행:

- `/admin/ingestion-jobs` 상단에서 `수집 실행` 버튼으로 사이트별 실행 가능
- 실행 옵션: 사이트 선택, `limit`, `dry-run`
- 푸시 알림 실제 전송 구조는 [푸시_알림_설계](/Users/shin/workspace/ArtTomato/docs/푸시_알림_설계.md) 기준으로 2차 반영

### 4.1 대구미술관(daegu-art) 옵션 운영

- 기본 정책: 비활성(운영 기본 대상 11개 유지)
- 포함이 필요하면 `INGESTION_ENABLE_DAEGU_ART=true`로 옵션 활성
- DNS/네트워크가 불안정하면 `INGESTION_DAEGU_USE_EDGE_FETCH=true`로 Edge fallback 사용
- 상세 절차: [대구미술관_외부수집_운영체크리스트](/Users/shin/workspace/ArtTomato/docs/대구미술관_외부수집_운영체크리스트.md)

외부 점검 커맨드:

```bash
npm run supabase:functions:deploy:daegu-test
npm run supabase:functions:invoke:daegu-test
npm run ingest:daegu:audit:edge
npm run ingest:daegu:dry-run:edge
```

직접 플래그를 지정해 실행할 수도 있다:

```bash
INGESTION_ENABLE_DAEGU_ART=true npm run ingest:audit-sites -- --site=daegu-art --sample=2
INGESTION_ENABLE_DAEGU_ART=true npm run ingest:run -- --site=daegu-art --dry-run --limit=20
```

시작 알림 배치 수동 실행:

```bash
npm run alerts:start-run -- --dry-run
npm run alerts:start-run
```
